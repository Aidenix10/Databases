Troubleshooting SQL Queries

Understanding the difference between Syntax Errors and Runtime Errors in data retrieval.

Based on OCR Document 5.2

Runtime Logic Errors

The code is written with valid words, but the logic is impossible for the database to execute.

Error: Column is invalid in the select list because it is not contained in either an aggregate function or the GROUP BY clause.

Syntax Errors

The code violates the grammatical rules of the language (spelling or structure).

Error: Incorrect syntax near 'GROUPBY'.

Scenario 1: The Aggregation Rule

When using GROUP BY, every column in your SELECT statement must be either aggregated (like COUNT) or listed in the group clause.

❌ Runtime Error 
SELECT LastName, FirstName, COUNT(OrderID)
FROM Customers
GROUP BY LastName
HAVING COUNT(OrderID) >= 5

Why it fails: The database doesn't know which "FirstName" to display for the "LastName" group if there are multiple people with the same last name but different first names.

✅ Corrected Code 
SELECT LastName, FirstName, COUNT(OrderID)
FROM Customers
GROUP BY LastName, FirstName
HAVING COUNT(OrderID) >= 5

The Fix: Add FirstName to the GROUP BY clause. Now we are grouping by unique pairs of names.

Scenario 2: Exact Spelling

SQL keywords cannot be combined. Spaces matter.

❌ Syntax Error 
SELECT LastName, FirstName
FROM Customers
GROUPBY LastName, FirstName

Why it fails: "GROUPBY" is not a word in the SQL dictionary. The parser stops here.

✅ Corrected Code 
SELECT LastName, FirstName
FROM Customers
GROUP BY LastName, FirstName

The Fix: Add a space. GROUP BY is a two-word clause.

Command Dictionary

Command / Term Description Key Rule
GROUP BY Groups rows that have the same values in specified columns into summary rows. Must include all non-aggregated SELECT columns.
HAVING Used to filter groups based on a condition (usually involves an aggregate). Applied after grouping. Does NOT require a WHERE clause.
COUNT() An aggregate function that returns the number of rows that match a specified criterion. Does not need to be in the GROUP BY clause.
Semicolons (;) Standard character to terminate a SQL statement. Often optional in single-statement environments (distractor in PDF).
Schema The container/namespace for database objects (e.g., dbo.Customers). Not required if the default schema is implied.
