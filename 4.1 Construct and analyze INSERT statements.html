Construct & Analyze INSERT Statements

A guide to SQL data insertion, row value constructors, and data migration strategies.

1. Core Concepts & Definitions

INSERT INTO

The primary SQL command used to add new rows of data into an existing database table.

Row Value Constructors

A method to specify multiple rows of data in a single VALUES clause. This allows you to insert batch data efficiently without writing multiple INSERT statements.

Subquery Insertion

Using a SELECT statement inside an INSERT statement to move data dynamically from a source table to a target table.

Target vs. Source

  • Target: The table receiving the data (e.g., INSERT INTO target...).
  • Source: The table providing the data (e.g., SELECT * FROM source...).

A. Single Row Insertion

The standard method to add one record. You must match the columns listed to the values provided.

INSERT INTO Cookies (Name, CookieID)
VALUES ('Snickerdoodle', 77);
Rule: The order of values in VALUES() must strictly match the order of columns specified after the table name.

B. Multi-Row Insertion

Using Row Value Constructors to insert multiple records at once.

INSERT INTO table_name (col1, col2)
VALUES 
  (value1a, value1b),
  (value2a, value2b);

Result: Creates two distinct rows in the database table.

3. Copying Data: The Distinction

Important

A common confusion in SQL is distinguishing between appending data to an existing table versus creating a new table from data. The PDF explicitly contrasts these two methods.

Feature INSERT INTO ... SELECT SELECT ... INTO
Purpose Copies data into an existing table. Copies data into a new table.
Does Table Exist? YES. The table must exist beforehand. NO. SQL creates it during execution.
Syntax INSERT INTO target
SELECT * FROM source; 		SELECT * INTO target
FROM source;
Keyword Usage Does NOT use the word VALUES. Uses the INTO keyword immediately after SELECT *.

4. Practical Scenario Analysis

Scenario: Promoting Customers

Goal: Move all records from potential_customers to the existing customers table.

Correct Syntax:

INSERT INTO customers
SELECT * FROM potential_customers

Incorrect: INSERT INTO customers VALUES SELECT... (Do not use VALUES!)

Scenario: New Cookie Product

Goal: Add "Snickerdoodle" with ID 77 to the Cookies table.

Correct Syntax:

INSERT INTO Cookies (Name, CookieID)
VALUES ('Snickerdoodle', 77)
