The "Party" Analogy

Joins connect lists side-by-side (adding columns).

Inner Join

Exclusive

Returns rows when there is at least one match in both tables.

Note: JOIN and INNER JOIN are the same.

Analogy: The Mutual Friends Party. You only get in if you are on BOTH lists.

Left Join

Keep everything from the Left list.

Analogy: Left side is VIP. They get in no matter what. Right side only gets in if they are with a Left VIP.

Right Join

Keep everything from the Right list.

Analogy: Right side is VIP. They get in no matter what.

Full Outer Join

Inclusive

Returns rows when there is a match in one of the tables.

Analogy: Everyone is invited. If you are on List A OR List B (or both), you get in.

Self Join

Comparing a list to itself.

Analogy: Looking for connections inside the same group (e.g., Finding students with the same birthday in one class).

Cartesian Product

Every possible combination.

Analogy: The "Outfit Mix & Match." 3 shirts x 2 pants = 6 outfits.

The "Lego" Analogy

Sets stack lists top-to-bottom (adding rows).

Command Logic Analogy
Union Combine + Remove Duplicates Dump two buckets of Legos into one pile. Remove identical duplicates.
Union All Combine + Keep Everything Dump two buckets and keep the pile exactly as is (even duplicates).
Intersect Match + Remove Duplicates Find the bricks that exist in BOTH buckets. Discard unique ones.
Intersect All Match + Keep Duplicates Same as Intersect, but if you have 2 matches, you keep 2 matches.

The "Filter" Analogy

Think of a coffee filter, or filtering "Shoes under $50" on Amazon.

The WHERE Clause

Definition

"The WHERE clause allows you to obtain a subset of rows."

"The WHERE clause is used to obtain only the records that fulfill the criterion in the WHERE clause."

Code Example SELECT * FROM Songs WHERE Genre = 'Rock'

Translation: "Show me the list of songs, but FILTER it to ONLY show me the Rock songs."

Rules & Limits

Keywords that clean up your results or set rules for data entry.

DISTINCT

The "De-Clutterer"

Returns each combination of matching values only once.

SELECT DISTINCT flowername FROM flowers

UNIQUE

The "Bouncer" (Rule)

Used to ensure a value is unique when entering data into a field.

TOP 1

The "Winner"

Used to return the first value of a record set.

Data Removal

Be careful! These commands destroy data.

DELETE

DML (Slow/Safe)

Removes records (rows) in a table.

Key Feature: You CAN use a WHERE clause.

Analogy: The Surgeon. Goes row by row, carefully removing specific parts. It logs every cut it makes.

TRUNCATE TABLE

DDL (Fast/Destructive)

Removes ALL data from a table without logging individual deletions.

Key Feature: You CANNOT use WHERE. It deletes everything.

Analogy: The Bulldozer. It doesn't check the rows; it just destroys the whole page and gives you a blank one.

Myth Buster: "REMOVE"

FALSE STATEMENT: "REMOVE marks the contents of a table for deallocation."

"REMOVE" is not a standard SQL command. Use DELETE or TRUNCATE instead.

Quick Decision Guide

Don't overthink it. Just follow the guide.

Need a Clean Mailing List?

UNION

Merges lists and removes duplicates.

Need a Bank Statement?

UNION ALL

Keeps every single record, even duplicates.

Need Common Ground?

INTERSECT

"Who bought an iPad AND an iPhone?"

Need Specific Items?

WHERE

"Show me only the red shoes."

Need to Clear Everything?

TRUNCATE

Wipes the table clean instantly. (Use DELETE for specific rows).
